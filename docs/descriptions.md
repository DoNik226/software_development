# Перечень тестов
## Класс Bonus
### 1. Метод update(self)
#### Тест №1.1 (позитивный)
* Цель: Проверить корректное завершение действия бонуса после окончания отведённого времени.
* Входные данные: Текущее время превышает начальное время появления бонуса на период равный или больший продолжительности бонуса.
* Ожидаемый результат: Свойство active должно стать равно False.
#### Тест №1.2 (негативный)
* Цель: Проверить сохранение активности бонуса, если срок его действия не завершён.
* Входные данные: Текущее время меньше установленного времени жизни бонуса.
* Ожидаемый результат: Свойство active должно остаться равным True.

## Класс Enemy
### 1. Метод move(self)
#### Тест №1.1 (позитивный)
* Цель: Проверить нормальное перемещение вражеского объекта внутри границ экрана.
* Входные данные: Координаты и размеры экрана, начальная позиция объекта и скорость движения.
* Ожидаемый результат: Объект перемещается в пределах экрана согласно своему направлению и скорости.
#### Тест №1.2 (негативный)
* Цель: Проверить невозможность выхода объекта за пределы экрана.
* Входные данные: Начальная позиция объекта вблизи границ экрана и высокая скорость движения.
* Ожидаемый результат: После нескольких шагов объект отразится от края экрана и продолжит двигаться внутрь.
#### Тест №1.3 (негативный)
* Цель: Проверить защиту от аномалий (нулевое или крайне низкое значение скорости).
* Входные данные: Очень низкая или нулевая скорость движения.
* Ожидаемый результат: Объект либо совсем не двигается, либо медленно перемещается.

## Класс Records
### 1. Метод save_record(self, data)
#### Тест №1.1 (позитивный)
* Цель: Проверить успешное сохранение одиночного рекорда.
* Входные данные: Правильное заполненное значение реквизитов.
* Ожидаемый результат: Рекорд успешно сохраняется в файл.
### 2. Метод load_records(self)
#### Тест №2.1 (позитивный)
* Цель: Проверить загрузку ранее записанных рекордов.
* Входные данные: Файл с одним правильным рекордом.
* Ожидаемый результат: Правильно считанный список с единственным элементом.  
#### Тест №2.2 (негативный)
* Цель: Проверить корректную обработку пустой базы рекордов.
* Входные данные: Пустой файл.
* Ожидаемый результат: Возвращается пустой список.  
#### Тест №2.3 (негативный)
* Цель: Проверить правильную обработку несуществующего файла.
* Входные данные: Название файла, которого нет.
* Ожидаемый результат: Вернуть пустой список без исключения.

### 2. Метод spawn_enemy(self, player_position)
#### Тест №2.1 (позитивный)
* Цель: Проверить, что враги появляются на безопасном расстоянии от игрока.
* Входные данные: Позиция игрока [400, 300].
* Ожидаемый результат: Дистанция между игроком и заспауненным врагом ≥ 170 пикселей.

### 3. Метод check_collision(self, player_rect)
#### Тест №3.1 (позитивный)
* Цель: Проверить уменьшение количества жизней при столкновении с врагом.
* Входные данные: Игрок с 3 жизнями сталкивается с одним врагом.
* Ожидаемый результат: Количество жизней уменьшается до 2.

#### Тест №3.2 (позитивный)
* Цель: Проверить обработку нескольких последовательных столкновений.
* Входные данные: Игрок с 3 жизнями сталкивается с 3 врагами подряд.
* Ожидаемый результат: Количество жизней становится равным 0.

### 4. Метод format_time(self, seconds)

### 6. Метод game_logic(self, current_time, player_position, player_speed)

#### Тест №6.2 (позитивный)
* Цель: Проверить спаун врагов через заданные интервалы времени.
* Входные данные: Интервал спауна 1000 мс, прошло 2000 мс.

#### Тест №6.3 (позитивный)
* Цель: Проверить обработку столкновения с врагом в игровой логике.
* Входные данные: Игрок сталкивается с врагом при 3 жизнях.
* Ожидаемый результат: Количество жизней уменьшается до 2, враг удаляется из списка.

#### Тест №6.4 (позитивный)
* Цель: Проверить завершение игры при потере всех жизней.
* Входные данные: Игрок с 1 жизнью сталкивается с врагом.
* Ожидаемый результат: Метод game_logic возвращает False.

#### Тест №6.5 (позитивный)
* Цель: Проверить защиту щита от столкновений с врагами.
* Входные данные: Активный щит и столкновение с врагом.
* Ожидаемый результат: Количество жизней не уменьшается, враг удаляется.

#### Тест №6.6 (позитивный)
* Цель: Проверить спаун бонусов через заданные интервалы времени.
* Входные данные: Прошло 8000 мс с последнего спауна бонуса.
* Ожидаемый результат: В списке бонусов появляется 1 бонус.

#### Тест №6.7 (позитивный)

* Входные данные: Игрок собирает бонус типа 'life' при 3 жизнях.
* Ожидаемый результат: Количество жизней увеличивается до 4, бонус удаляется.

#### Тест №6.8 (позитивный)

### 7. Метод run_game(self)
#### Тест №7.1 (позитивный)
* Цель: Проверить корректную инициализацию игры при запуске.
* Входные данные: Запуск игры с уровнем сложности "Легкий".
* Ожидаемый результат: Загружается изображение корабля, устанавливается начальное время, собираются результаты.

#### Тест №7.2 (позитивный)
* Цель: Проверить завершение игры при game over.
* Входные данные: Игрок теряет все жизни.
* Ожидаемый результат: Игра завершается, собираются результаты с продолжительностью ≈0.1 сек.

#### Тест №7.3 (позитивный)
* Цель: Проверить корректный расчет времени игры.
* Входные данные: Игра выполняется 4 секунды.
* Ожидаемый результат: В результатах продолжительность равна 4 секундам.

#### Тест №7.4 (позитивный)
* Цель: Проверить корректность параметров для разных уровней сложности.
* Входные данные: Создание игр с уровнями "Легкий", "Средний", "Сложный".
* Ожидаемый результат: Соответствующие значения жизней и интервалов спауна.

#### Тест №7.5 (негативный)
* Цель: Проверить обработку исключений во время выполнения игры.
* Входные данные: Исключение "Test exception" во время обработки событий.
* Ожидаемый результат: Исключение пробрасывается дальше.

#### Тест №7.6 (позитивный)
* Цель: Проверить корректное использование ресурсов при запуске игры.
* Входные данные: Запуск игры с последующим завершением.
* Ожидаемый результат: Все необходимые методы вызываются однократно.

* Ожидаемый результат: Имя в результатах пустое, player_name устанавливается в пустую строку.

#### Тест №8.3 (позитивный)
* Цель: Проверить сбор результатов для разных уровней сложности.
* Входные данные: Игры с уровнями "Легкий", "Средний", "Сложный".
